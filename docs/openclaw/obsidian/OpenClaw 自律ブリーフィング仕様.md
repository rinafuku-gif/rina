# OpenClaw 自律ブリーフィング仕様

> `scripts/openclaw-morning-briefing.sh` の設計思想と動作仕様。

## コンセプト

従来の朝ブリーフィングは**固定テンプレート**でフォーマットが決まっていた。
OpenClaw版は **Claude 自身が状況を判断して、内容・構成・優先順位を決める**。

## 実行フロー

```
毎朝 7:00（launchd）
  ↓
openclaw-morning-briefing.sh
  ├─ git pull origin main（リポジトリ最新化）
  ├─ Airbnb予約同期（しらたまサーバーが動いていれば）
  ├─ claude -p で自律ブリーフィング生成（7分タイムアウト）
  │   ├─ Google Calendar（今日〜3日先）を読む
  │   ├─ CLAUDE.md のタスク管理セクションを読む
  │   ├─ logs/daily/ の直近ログを読む
  │   ├─ docs/ 配下を必要に応じて読む
  │   ├─ 曜日・状況に応じて内容を自律構成
  │   └─ logs/daily/ にログ記録 → git commit & push
  └─ 生成されたブリーフィングを LINE Push で送信
```

## 自律判断の指針

Claude に渡しているガイドライン:

### 優先順位の判断
- **物理的にやるべきこと** を最優先（ゲスト対応、打ち合わせ、立ち会い等）
- 締め切りが近いタスクがあれば警告
- 予約がある日は準備リマインド

### 曜日ごとのアプローチ
- **月曜**: 今週の見通しを含める
- **金曜**: 週末の予定・来週の準備事項
- **それ以外**: その日に集中すべきことにフォーカス

### 情報量の自動調整
- 忙しい日 → 重要なことだけ簡潔に
- 穏やかな日 → 進めるべき中長期タスクを提案
- 前回「やる」と言って未完了のもの → さりげなくリマインド

## 出力ルール

- LINE で読みやすい形式（短文、箇条書き中心）
- 冒頭: 「おはよう、Ryo。」で統一
- セクション見出し: 【】で括る
- 最大500文字目安

## 情報ソース

| ソース | 読み取り内容 |
|---|---|
| Google Calendar（全カレンダー） | 今日〜3日先の予定 |
| `CLAUDE.md` | タスク管理セクション、事業優先順位 |
| `logs/daily/` | 直近のログから継続事項・未完了事項 |
| `docs/` 配下 | 各事業の最新ドキュメント（必要に応じて） |

## 環境変数（.env）

```
LINE_CHANNEL_ACCESS_TOKEN   # LINE Push 送信
LINE_USER_ID                # Ryo の LINE ID
SHIRATAMA_API_TOKEN         # Airbnb同期API認証
```
