# しらたまタスク抽出アーキテクチャ v2

## 設計日: 2026-03-18
## ステータス: 設計中

---

## 1. 現状の問題

- tasks.json はCLAUDE.mdから1回移行しただけの静的データ
- 情報ソースは繋がっているが横断的にタスクを抽出していない
- 「今日何をすべきか」に答えられない

## 2. 設計思想

**タスクは「作るもの」ではなく「発見するもの」**

Ryoが手動でタスクを登録するのではなく、
既存の情報ソースから自動的にタスクを「発見」する。

### タスクの発生パターン

| パターン | 例 | ソース |
|---------|-----|--------|
| 予定がある | 明日Airbnbゲスト到着 | Calendar + Bookings |
| 期限がある | 補助金申請 4/30まで | Calendar + Drive |
| 作業が進行中 | しらたまPWA改善 | Git commits |
| 誰かが待っている | ゲストからのメッセージ | Airbnb (Gmail) |
| 定期的にやるべき | 経費処理、SNS投稿 | スケジュールルール |
| AIが提案 | 「この契約書そろそろ確認した方がいい」 | ブリーフィング |

## 4. 各ソースからの抽出ロジック

### 4a. Google Calendar → 「今日の予定 + 準備タスク」
- 今日〜3日先の予定を取得
- 予定に応じた準備タスクを生成:
  - Airbnbゲスト到着 → 「部屋の準備」「チェックイン時間確認」
  - MTG → 「議題確認」「資料準備」
  - 三十日珈琲出店 → 「豆の準備」「機材チェック」

### 4b. Git Repos → 「デジタル作業の進捗」
- 直近のコミットから何が進んだかを検知
- 長期間コミットがないプロジェクト → 「放置されてるけど大丈夫？」
- デプロイ失敗 → 緊急タスク

### 4c. Gmail → 「対応が必要なメール」
- Airbnb: 新規予約、キャンセル、ゲストメッセージ
- その他: 返信が必要なメールの検知（未読 + 重要ラベル）

### 4d. Notion → 「DX案件の進捗」
- DX案件DBをクエリ → ステータスが「対応中」のもの
- 教材DB → 下書き状態のコンテンツ
- テンプレートDB → 未出品のテンプレート

### 4e. Google Drive → 「期限のある書類」
- 特定フォルダ（契約書、補助金）の更新状況
- deadlines.json との照合

### 4f. Airbnb Bookings → 「ゲスト対応」
- 今日〜3日先のチェックイン/チェックアウト
- レビュー返信の必要性
