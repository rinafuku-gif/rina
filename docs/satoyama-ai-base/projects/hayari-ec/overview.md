### 顧客管理 + メルマガ（設計中）

#### 要件

- 無料（または極めて低コスト）
- 村上さん自身で運用できるシンプルさ
- 他の発信チャネル（SNS等）とシームレスに連動
- 顧客情報のセキュリティが明確

#### 推奨構成: Brevo（旧Sendinblue）

| 項目 | 内容 |
|---|---|
| サービス | Brevo（brevo.com） |
| 無料枠 | 連絡先数 無制限、1日300通（月 約9,000通） |
| 機能 | メルマガ配信、顧客リスト管理、配信テンプレート、開封率トラッキング |
| セキュリティ | GDPR準拠、データはEU/USサーバー、2FA対応 |
| 運用 | ブラウザで管理画面にログイン → テンプレ選択 → 配信 |
| API連携 | Next.jsから購入時に自動で顧客リストに追加可能 |

**Brevoを推奨する理由:**
1. **連絡先数が無制限**（Mailchimpは500件制限）— ECは顧客が増えるほど価値が出るので件数制限はキツい
2. **CRM機能が内蔵** — 購入履歴・配信履歴を一元管理
3. **テンプレートエディタ** — 非エンジニアでもドラッグ&ドロップでメルマガ作成
4. **API連携** — Stripe決済完了 → Brevo顧客リスト自動登録の導線が組める
5. **セキュリティ** — GDPR準拠で顧客データの取り扱い方針が明確

#### 代替案

- **Resend + Google Sheets**: より技術寄り。APIは洗練されているがCRM機能なし。村上さんの自走には不向き
- **Mailchimp**: UIは良いが無料枠が500件と小さい。成長するとすぐ有料化

## セキュリティ方針

### 顧客情報の取り扱い

| データ | 保管場所 | 暗号化 | アクセス |
|---|---|---|---|
| クレジットカード情報 | Stripe側で保管（サイト側に一切保存しない） | PCI DSS準拠 | Stripe経由のみ |
| 氏名・住所・メール | Brevo（メルマガ）+ Stripe（決済） | SSL/TLS暗号化 | 管理者のみ |
| 注文履歴 | Stripeダッシュボード | SSL/TLS暗号化 | 管理者のみ |
| パスワード | なし（会員登録不要の設計） | — | — |

### ポイント

- **カード情報はサイト側に一切保存しない** — Stripe Checkout or Stripe Elements を使用し、カード情報はStripeのサーバーで直接処理
- **会員登録不要** — メールアドレスのみで購入可能。パスワード漏洩リスクをゼロに
- **SSL必須** — 全ページHTTPS化
- **プライバシーポリシーに明記** — 取得する情報、利用目的、第三者提供の有無、問い合わせ先
