# "あの業務、もっと楽にならない？"を90分で解決する会

— 生成AIとITツールで、今の環境からできる業務改善を一緒に見つけます

## 基本情報

- 日時: 2026年3月13日（金）
- 場所: アンニャモンニャ
- 時間: 15:00〜16:30（90分）
- 定員: 6名
- 参加費: ドネーション制（1,000円〜）
- 対象: AI触ったことある〜使いこなしたい層
- 持ち物: スマホまたはノートPC
- 使用ツール: Gemini（無料）
- 主催: SATOYAMA AI BASE（稲福 良祐）

## コンセプト

**「あなたの業務課題をAIやITツールを使って解決するワークショップ」**

参加者はそれぞれ自分の事業の困りごとを持ち寄る。AIやITツールはあくまで道具。大事なのは：

1. 何を実現したいのか（ゴールの明確化）
2. 今の業務の何がめんどくさいのか（課題の言語化）
3. 普段使っている環境は何か（スマホ、PC、使ってるアプリ等）
4. 今ある環境だけで実現できるか（新しいツールや費用をかけずに）

この4つを整理した上で、AIに「手順書」を作らせ、その場で一番簡単な部分を実装して帰る。
残りは同じ要領で自分で進められる状態にする。

## 設計思想

- 参加者はソロ事業者（飲食・セラピスト等）。大きく稼げていない層
- 無料 or 今あるものだけで完結する提案が絶対条件。高いツールの導入は提案しない
- 知識を教えるのではなく、自分の課題を自分で解決できた体験を持ち帰ってもらう
- 「AIの使い方」ではなく「AIへの伝え方（プロンプト）」をレクチャーする
- 全部を90分で解決するのは無理 → Ryoがヒアリングして今日やる範囲を切り出す
- **「見せてからやらせる（Show Then Do）」が鉄則。** デモ→実践→共有のサイクルを回す
- **ユースケースは3つ以下に絞る。** 広く浅くより、深く体験させる方が持ち帰り度が高い
- **最初の5分で全員が発言する状態を作る。** 研究では最初の数分で発言しない人はその後も発言しにくくなる

## タイムテーブル（90分）

### 1. アイスブレイク & ゴール共有（7分）[0:00-0:07]

**目的: 全員が声を出す + 今日のゴールを揃える**

- Ryo の自己紹介（30秒。「AIの専門家」ではなく「AIを使って自分のめんどくさいを潰してる人」として）
- **アイスブレイク「AIに自己紹介させてみよう」:**
  - 全員にGeminiを開いてもらう（ここでアカウント・ログイン確認も兼ねる）
  - 「私は○○をやっています。私の事業を50字で紹介して」と打ってもらう
  - AIが生成した自己紹介を一人ずつ読み上げ → 笑いが起きやすく、AIの力を体感する一石二鳥
- 今日のゴール: **「業務の"めんどくさい"を一つ、楽にして帰る」**
- 前提共有: 「今日は"AIの勉強会"ではなく"業務改善ワークショップ"です」

> **進行メモ:** アイスブレイクでGeminiを開かせることで、ログイン確認と最初の成功体験を同時に済ませる。ここで全員がAIと初対話を済ませておくことが後半の活発さに直結する。

### 2. デモ: AIに仕事を伝えるコツ（13分）[0:07-0:20]

**目的: 「伝え方で結果が変わる」を体感させる**

#### デモ1: ダメな例 → いい例（5分）

PC画面をみんなに見せながら実演。

- **ダメな例:** 「メールを書いて」→ 汎用的でつまらない結果が出る
- **いい例:** 「あなたは山梨の古民家民泊のオーナーです。明日チェックインする海外ゲストに、到着時の案内メッセージを英語で送りたい。最寄り駅（鳥沢駅）からの道順と、鍵の受け取り方法を含めて、フレンドリーな口調で書いてください」→ すぐ使えるメールが出てくる

**AIにうまく伝えるための3つのポイント（レシピ）:**

| ポイント | 内容 | 言い換え |
|---------|------|---------|
| 1. WHO | 自分が何者かを伝える | 「私は○○です」 |
| 2. WHAT + FOR WHOM | ゴールと相手を具体的に | 「○○向けに、○○を作って」 |
| 3. HOW | 条件・制約を伝える | 「○○な口調で、○○文字以内で」 |

「この3つを入れるだけで、AIの回答が劇的に変わります」

#### デモ2: Gemini Live — 話しかけるだけ（3分）

**スマホだけの参加者にとって最大のaha moment**

- Geminiアプリの音声機能で、話しかけるだけでAIが応答するデモ
- 「キーボードで打たなくても使えるんです」
- 例: 「今日の仕事で一番めんどくさいことを相談したいんだけど...」と話しかける

> **進行メモ:** Gemini Liveのデモは「AIは難しい」という先入観を一瞬で壊す。スマホに話しかけるだけで自然に会話できる体験は、特にPC操作が苦手な参加者に刺さる。

#### デモ3: 画像も作れる（2分）

- 「今度の出店用のInstagram投稿画像を作って。カフェのラテアートの写真風で」
- Geminiの画像生成（無料で1日20枚）でSNS投稿用のビジュアルを生成
- 「デザインツールを使えなくても、こういうことができます」

#### ミニ実践（3分）

デモを見た直後に参加者にやってもらう:
- 「さっきの3つのポイントを使って、自分の事業のメール or メッセージを1つ書かせてみましょう」
- 隣の人と結果を見せ合う（ペアシェア30秒）

### 3. 課題ヒアリング & マッチング（15分）[0:20-0:35]

**目的: 各自の「めんどくさい」を言語化し、今日やる範囲を切り出す**

#### ペアインタビュー（6分）

2人1組で以下を聞き合う（3分×2）:
1. 何を実現したい？
2. 今、何がめんどくさい？
3. 普段使ってるツールは？（スマホ？PC？LINE？Googleアプリ？紙？）

#### 全体共有 & 今日のスコープ決め（9分）

- 各ペアから相手の課題を紹介（1ペア1.5分×3ペア = 4.5分）
- Ryoがその場で「今日できる範囲」を切り出す（4.5分）
  - 「全部は今日やれないけど、まずこの部分だけやってみましょう」
  - 今の環境で実現可能か、新しいツールが必要かもこの時点で判断

> **進行メモ:** 自分で「めんどくさい」を言うより、他人に紹介されると客観的になれる。またペアで聞き合うことで全員が喋る機会が生まれ、Ryoだけが聞き出す構図を避けられる。似た課題のペアをこの時点で見つけておくと後半のサポートが効率的。

### 4. 実践: AIで解決してみよう（30分）[0:35-1:05]

**目的: 自分の課題を自分でAIに投げ、動くものを1つ作る**

#### ステップ1: プロンプトを書く（8分）

- さっき言語化した課題を、デモで見せた3つのポイントに沿ってプロンプトにする
- AIに投げる: 「この課題を解決するための手順書を、私のスキルレベルに合わせて作って」
- 出てきた手順書の中から、**今日この場で10分でできるもの**を1つ選ぶ

**声かけ例:**
- 「まず打ってみて、変な回答が出ても大丈夫。それを直すのも練習です」
- 「完璧じゃなくていい。AIは何回でもやり直してくれるから」

#### ステップ2: 手順書の最初の一歩を実装する（15分）

手順書の中から最も簡単な部分を今ここで実装する。

- Ryoが巡回して個別にサポート
- 詰まったらまずAIに聞く → これ自体が「AIの使い方」の練習
- **早く終わった人は「隣の人を手伝う」ミニメンター役に**

#### ステップ3: 発展課題（7分）

基本が終わった人向け:
- 「さっき作ったものを、もう1パターン作ってみよう」
- 「AIに『もっと良くして』と頼んでみよう」
- 「Gemini Liveで音声で指示してみよう」

**まだ基本に取り組んでいる人:**
- Ryoが重点サポート
- 「ここまでできたら十分。残りは家でもできるから」と安心感を与える

> **進行メモ:** この30分が一番重要。Ryoの巡回で「課題の切り出し」がうまくいっているかを確認する。全員が「何かしら動くもの」を作れるよう、課題が大きすぎる場合は迷わず範囲を狭める。AIが変な回答を出したら「こうやって修正を頼めばいい」と見せる → 失敗も学びにする。

### 5. 成果シェア & ネクストステップ（15分）[1:05-1:20]

**目的: 成功体験を言葉にする + 明日からの行動を決める**

#### 成果発表（8分）

- 各自1分で今日やったことをシェア（1分×6人）
- 「何を作ったか」「どれくらい時間が短縮されそうか」
- 他の人の活用法を聞くことで「自分もそれやりたい」が生まれる

#### 「明日やる1つ」を決める（3分）

- 各自、今日作った手順書の中から**明日やる次の1ステップ**を決めて宣言する
- 紙に書いてもらうか、スマホのメモに残してもらう

#### フォローアップ案内（4分）

- **持ち帰り:**
  - 今日作ったプロンプト＆手順書はそのまま自分のGeminiに残っている
  - プロンプトチートシート（A4両面1枚 → 事前準備で印刷）
  - 困ったらAIに聞く。「わからない」と正直に伝えれば噛み砕いて教えてくれる
- **SATOYAMA AI BASEの紹介（押し売りしない）:**
  - 「もっと深くやりたい人向けに、個別の相談もやっています」
  - LINE公式の友だち追加を案内（QRコード表示）
  - 「来週、15分のオンラインQ&Aを開きます。つまづいたことがあれば気軽にどうぞ」

### 6. 自由交流タイム（10分）[1:20-1:30]

- 質問、雑談、個別相談
- 参加者同士の名刺交換やつながり作り
- **フィードバック収集:**
  - 「一言だけ教えてください。今日来て、一番良かったことは？」（口頭でOK）
  - 余裕があれば15秒の動画感想をお願いする（テキストの10倍の訴求力）

## Ryoの進行メモ（重要）

### 言葉遣い

| 使わない言葉 | 代わりに使う言葉 |
|-------------|----------------|
| プロンプト | AIへの指示文、AIへの頼み方 |
| プロンプトエンジニアリング | AIにうまく伝えるコツ |
| LLM、大規模言語モデル | AI、AI会話ツール |
| トークン | （言及しない） |
| ハルシネーション | 「AIが自信満々に間違えること。よくあるので気にしないで」 |
| 生成AI | AI（必要なら「文章や画像を作れるAI」） |

### AIの説明に使えるたとえ話

| 概念 | たとえ |
|------|-------|
| AIの能力 | 「何でも知ってる新入社員。めちゃくちゃ詳しいけど、指示が曖昧だと見当違いのことをやる」 |
| プロンプトの大事さ | 「初めて来るバイトに仕事を教えるとき、丁寧に説明するほどちゃんとやってくれるのと同じ」 |
| AIの間違い | 「テスト中にわからない問題を堂々と間違える優等生。自信があるように見えても必ず自分で確認する」 |

### ペース配分のコツ

- **15分ごとにアクティビティを切り替える。** 90分の集中力の限界を意識し、リズムを作る
- **説明→実践→共有のサイクル**を3回転がベスト
- 実装タイムの巡回では「できた人」より「止まっている人」を先にケアする
- 早く終わった人は「隣の人を手伝ってもらえますか？」とミニメンター化

### ライブデモのリスク対策

| リスク | 対策 |
|-------|------|
| AIが変な回答を出す | 「これ、ちょっと微妙ですね。こうやって修正します」→ 失敗を教材にする。事前に「AIは毎回違う答えを出すので、変なのが出ても全然OK」と前フリ |
| Wi-Fiが不安定 | モバイルホットスポットを持参。事前にアンニャモンニャのWi-Fi速度を確認 |
| 参加者がログインできない | Geminiアカウント作成の手順メモを印刷しておく。Googleアカウントがない人用にその場で作成サポート（最大5分想定） |
| 全員のペースが合わない | レイヤード課題（基本→発展）で吸収。早い人はミニメンター役に |

### 参加者への声かけフレーズ集

| 場面 | フレーズ |
|------|---------|
| 最初の一歩が踏み出せない人 | 「まず何でもいいから打ってみましょう。変な答えが出ても全然大丈夫」 |
| AIの結果が微妙だった時 | 「いいですね！あとは『もうちょっと○○にして』って言えばOK」 |
| すごく良い結果が出た時 | 「おお、これすごくいい！○○さん、みんなに見せてもらっていいですか？」 |
| 時間が足りない人 | 「ここまでで全然十分。残りは同じやり方で家でもできますから」 |
| 操作に不安がある人 | 「スマホに話しかけるだけでも使えるので、Gemini Liveも試してみましょう」 |
