# DXヒアリングフォーム 設計書

> 目的: Notionフォームで顧客からヒアリングした情報をもとに、AIが自律的にプロジェクトを進められる状態を作る
> 更新日: 2026-03-15

## 設計方針

1. **顧客が迷わない**: 選択式中心。自由記述は最小限
2. **相談内容で分岐**: ECなのか業務改善なのかでフォームの質問を出し分ける（Notion条件付き表示）
3. **AIが判断できる**: フォーム回答だけで技術スタック・見積もり・スケジュールを自動生成できる粒度
4. **二度手間にしない**: フォームで聞けることは後から電話で聞き直さない

## 条件付き表示ルール（Notionフォーム）

```
ご相談内容 = "業務改善・効率化"
  → セクション4を表示

ご相談内容 = "ECサイト構築"
  → セクション5を表示

ご相談内容 = "Webサイト制作"
  → セクション6を表示

ご相談内容 = "SNS運用・発信"
  → セクション4の一部（ラクにしたい作業、頻度、一番嬉しいこと）を表示

ご相談内容 = "その他"
  → セクション4を表示（汎用的なので）
```

## AI自動処理フロー

```
フォーム送信
  ↓ Notion Webhook → Make → api.tonari2tomaru.com
  ↓
1. ヒアリング回答を構造化分析
   - 相談内容の分類確認
   - 技術的な実現方法の判断
   - 類似案件のパターンマッチ
  ↓
2. 自動生成物
   - 要件定義書（Markdown）
   - 技術スタック提案
   - 概算見積もり
   - スケジュール案
  ↓
3. Ryoに LINE通知
   「新規案件: [お名前] - [ご相談内容]」
   「提案書を生成しました → [URL]」
  ↓
4. Ryoが確認・修正 → 顧客に提出
  ↓
5. 契約後 → Claude Code CLIが実装開始
```
